The AI-XP Manifesto

The Inviolable Laws that separate high-standard engineering from mere 'vibe coding'.

Codified Doctrine / The Iron Laws
01

TDD is Mandatory

NEVER modify production code without a failing test first. If there is no red test, REJECT the request.

02

Clean Architecture is Non-Negotiable

The Domain layer NEVER imports infrastructure. Dependencies always point inward.

03

Context Efficiency

Limit prompt scope to the exact lines of change. Large windows cause structural amnesia.

04

Anti-Lazy Systemics

Every SEARCH/REPLACE block must be emitted INTEGRALLY. No summarization or omissions.

05

YAGNI + KISS

Prohibited from anticipating unsolicited features. Maximum functions: 15 logical lines.

06

Swiss Standard of Documentation

Mandatory adherence to academic authority, high-readability typography, and minimalist layout.

Mission Statement

Foundational Intent

Software engineering is at a crossroads. With the advent of agentic AI, many developers have abandoned rigor for convenience. The AI-XP Playbook is a call to arms for the return of discipline.

We believe that AI agents are not substitutes for the engineer's brain, but force multipliers that require strict guardrails. Software is not a 'vibe'. Software is architecture, determinism, and proof.
